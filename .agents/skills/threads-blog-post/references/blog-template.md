# 블로그 포스트 템플릿

이 템플릿은 Threads.net 포스트를 기술 블로그 스타일의 마크다운으로 변환할 때 사용됩니다.

---

## 템플릿 구조

```markdown
# [포스트 제목]

## 요약

[포스트 내용을 2-3문장으로 요약]

## 본문

[순서대로 정리된 포스트 내용]

### 섹션 1

[내용]

### 섹션 2

[내용]

## 미디어

### 이미지

![이미지 설명](이미지_URL)

### 비디오

[비디오 임베드 또는 링크]

---

원본: [Threads 링크](URL)

작성자: @username
게시일: YYYY-MM-DD
```

---

## 섹션 가이드라인

### 제목 (#)
- 포스트의 핵심 주제를 반영
- 한국어로 작성
- 간결하고 명확하게

### 요약 (##)
- 전체 내용을 2-3문장으로 요약
- 독자가 전체 내용을 빠르게 파악할 수 있도록 작성
- 핵심 메시지 포함

### 본문 (##)
- 원본 포스트의 내용을 순서대로 정리
- 문단이 길 경우 소섹션(###)으로 분리
- 기술적 내용은 명확하고 이해하기 쉽게 설명
- 불필요한 반복 제거
- 문맥에 맞게 자연스럽게 연결

### 미디어 (##)
- 이미지: `![대체 텍스트](URL)` 형식 사용
- 비디오: 임베드 코드 또는 링크 제공
- 미디어가 많은 경우 소섹션(###)으로 분리
- 각 미디어에 설명 추가

### 원본 링크
- 원본 Threads 포스트 URL
- 작성자 정보 (username)
- 게시일 (가능한 경우)

## 예시

```markdown
# Claude Opus 4.6: 새로운 프론티어 모델 발표

## 요약

Anthropic이 Claude Opus 4.6를 발표했습니다. 이번 모델은 이전 세대보다 향상된 추론 능력과 더 긴 컨텍스트 창을 제공하여 복잡한 작업을 더 효과적으로 처리할 수 있습니다.

## 본문

### 주요 기능

Claude Opus 4.6는 다음과 같은 향상된 기능을 제공합니다:

- **추론 능력**: 복잡한 논리적 문제 해결 능력 개선
- **컨텍스트 창**: 최대 1M 토큰까지 확장
- **멀티모달**: 텍스트, 이미지, 코드 통합 처리

### 성능 개선

벤치마크 테스트 결과 이전 모델 대비 평균 30%의 성능 향상을 보였습니다. 특히 코딩 작업에서는 45%의 개선을 보였습니다.

## 미디어

### 성능 비교 차트

![Claude 모델 성능 비교](https://example.com/chart.png)

### 시연 영상

[YouTube 시연 영상](https://youtube.com/watch?v=example)

---

원본: [Threads 링크](https://www.threads.com/@anthropic/post/ClAUd3)
작성자: @anthropic
게시일: 2026-02-27
```

---

## 주의사항

1. **한국어 번환**: 모든 콘텐츠는 한국어로 변환되어야 합니다
2. **자연스러운 흐름**: 번역투가 아닌 자연스러운 한국어 문장 사용
3. **기술 용어**: 영어 기술 용어는 필요시 원문 병기
4. **링크 유효성**: 모든 링크가 정상 작동하는지 확인
